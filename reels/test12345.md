# Automated Reels Editing & Upload Pipeline - Setup & Execution Guide

This document provides a comprehensive guide on how the pipeline works, where to acquire each required API key/credential, and how to run the pipeline.

---

## 🛠️ How It Works (Pipeline Workflow)

The pipeline is designed to run unattended on a schedule (e.g., hourly via Cron or Windows Task Scheduler) and process manually downloaded video clips.

```
[Manually Place Videos in raw/]
              │
              ▼
   (1. Scan & Register in DB)
              │
              ▼
   (2. FFmpeg quality enhancement)
              │
              ▼
   (3. Upload to Cloudflare R2)
              │
              ▼
   (4. Post to Instagram via Graph API)
```

1. **Scan `raw/` directory**: The orchestrator (`main.py`) scans the local `raw/` directory for any new video files (e.g. `.mp4`, `.mov`, `.avi`, `.mkv`).
2. **Duplication Guard**: It registers new files in `pipeline.db` (SQLite) to track them. It will never double-process or double-upload the same file.
3. **Enhancement & Transitions**: FFmpeg executes configured quality filters (denoising, deinterlacing, upscale to 1080x1920 with lanczos/crop, sharpening, color pop, 2-pass stabilization, and audio/video fade-ins/outs). The output is saved to `processed/`.
4. **Public Hosting**: The enhanced video is uploaded to Cloudflare R2 (or Firebase/Supabase) to obtain a public URL (required by Instagram Graph API).
5. **Captioning**: Captions are built automatically by pairing the video filename with niche-specific hashtag lists (`config.yaml`).
6. **Instagram Posting**:
   - Creates a Reels media container using the public video URL.
   - Polls container status until Instagram processing is `FINISHED`.
   - Publishes the container to your Instagram Business Feed.
7. **Constraints Guard**: Daily limits (max 25 uploads) and spacing constraints (e.g., 3 hours between posts) are checked before posting. In test mode, it stops after container validation and does not publish.

## 🔑 Where to Get Each API Key & Credential

### 1. Cloudflare R2 (S3-Compatible Hosting)
Instagram's Graph API requires a public URL to download the video. Cloudflare R2 offers a free tier (10 GB/month) with no egress fees.

1. Log into your [Cloudflare Dashboard](https://dash.cloudflare.com/).
2. Go to **R2 Object Storage** in the sidebar.
3. Click **Create Bucket**, name it (e.g., `reels-pipeline-bucket`), and create it.
4. Select the created bucket, go to the **Settings** tab:
   - Under **Public Access**, select **Custom Domains** or enable the **R2.dev Subdomain** (and copy the URL prefix, e.g. `https://pub-xxxxxx.r2.dev`).
5. Go back to **R2** > **Manage R2 API Tokens** (in the right panel of the R2 main screen).
6. Click **Create API Token**:
   - Permissions: **Read & Write** (or Edit).
   - Set the TTL to unlimited or a long duration.
7. Copy the credentials generated:
   - **Access Key ID** (stores as `R2_ACCESS_KEY_ID` in `.env`)
   - **Secret Access Key** (stores as `R2_SECRET_ACCESS_KEY` in `.env`)
   - **Account ID** (found on the R2 homepage or URL of your Cloudflare panel; stores as `R2_ACCOUNT_ID` in `.env`)

---

### 2. Instagram Graph API (Meta Developer App)
To post automatically via API, your Instagram account must be linked to a Facebook Page, and you need a Meta Developer App.

#### Step A: Link Account & Verify Type
1. Switch your Instagram account to a **Business Account** or **Creator Account** (via Instagram Mobile App Settings).
2. Create a **Facebook Page** (if you don't have one).
3. Link your Instagram Account to your Facebook Page:
   - Go to your Facebook Page > **Settings** > **Linked Accounts** > **Instagram** > Click **Connect Account**.

#### Step B: Create Meta Developer App
1. Go to the [Meta for Developers](https://developers.facebook.com/) portal.
2. Register and click **My Apps** > **Create App**.
3. Select **Other** > **Business** app type, name your app, and click **Create App**.
4. In the app dashboard, add the **Instagram Graph API** product.

#### Step C: Generate Instagram Business Account ID
1. Go to the [Graph API Explorer](https://developers.facebook.com/tools/explorer/).
2. Select your App in the top right.
3. Set permissions to include:
   - `instagram_basic`
   - `instagram_business_content_publish`
   - `pages_show_list`
   - `pages_read_engagement`
4. Click **Generate Access Token** and log in with your Facebook account.
5. In the Explorer URL box, enter:
   `me/accounts?fields=instagram_business_account{id,username},name`
6. Click **Submit**. Locate the nested `instagram_business_account` object for your page and copy the `id` string. This is your `INSTAGRAM_USER_ID`.

#### Step D: Generate a Long-Lived Access Token (60-Day)
Tokens generated in the Explorer tool expire in 1-2 hours. To generate a 60-day token:
1. Go to the [Access Token Tool](https://developers.facebook.com/tools/accesstoken/).
2. Locate your app and click **Need to extend this token?** beside the User Token.
3. Or make the following API request (using Postman or curl):
   ```
   GET https://graph.facebook.com/v19.0/oauth/access_token?  
       grant_type=fb_exchange_token&          
       client_id={your-app-id}&
       client_secret={your-app-secret}&
       fb_exchange_token={your-short-lived-token}
   ```
4. Copy the long-lived `access_token` and store it in `.env` as `INSTAGRAM_ACCESS_TOKEN`.

## 🚀 How to Run the Pipeline

### 1. Environment Setup
Make sure Python 3.11+ is installed.
1. Populate your keys inside [reels-pipeline/.env](file:///d:/Instagram/reels-pipeline/.env):
   ```ini
   R2_ACCOUNT_ID="your_cloudflare_account_id"
   R2_ACCESS_KEY_ID="your_access_key"
   R2_SECRET_ACCESS_KEY="your_secret_key"
   R2_BUCKET_NAME="reels-pipeline-bucket"
   R2_PUBLIC_URL_PREFIX="https://pub-xxxxxx.r2.dev"

   INSTAGRAM_USER_ID="your_instagram_business_account_id"
   INSTAGRAM_ACCESS_TOKEN="your_long_lived_token"

   # Uncomment if ffmpeg is not in system PATH
   # FFMPEG_PATH="C:/ffmpeg/bin/ffmpeg.exe"
   # FFPROBE_PATH="C:/ffmpeg/bin/ffprobe.exe"
   ```

---

### 2. Add Your Videos & Niches
1. Manually download your clips from the shared folder and save them into the `raw/` directory (e.g. `raw/marketing_tips.mp4`, `raw/finance_tips.mp4`).
2. Map your videos to specific niches in [reels-pipeline/niche_map.csv](file:///d:/Instagram/reels-pipeline/niche_map.csv):
   ```csv
   filename,niche
   marketing_tips.mp4,tech_ai
   finance_tips.mp4,finance
   ```
   *Any videos not mapped in the CSV will automatically fallback to the default niche (`lifestyle`) configured in `config.yaml`.*

---

### 3. Verify in Test Mode (Dry Run)
Ensure `test_mode` is set to `true` in [reels-pipeline/config.yaml](file:///d:/Instagram/reels-pipeline/config.yaml). This restricts the pipeline to 2 clips, prints FFmpeg command parameters, uploads them to R2, validates API access, but **stops before posting anything live**.

Run the orchestrator:
```bash
.venv\Scripts\python main.py
```
Check the output logs in `logs/main.log` and the edited video in `processed/test/`.

---

### 4. Switch to Production (Live Posting)
1. Edit [config.yaml](file:///d:/Instagram/reels-pipeline/config.yaml):
   ```yaml
   test_mode: false
   ```
2. Run again:
   ```bash
   .venv\Scripts\python main.py
   ```
   This will scan the `raw/` folder, register any new files, enhance them, upload them to R2, and publish the oldest clip in the queue to your Instagram feed.

---

### 5. Setup Automated Scheduling

#### Windows (Task Scheduler)
1. Search for **Task Scheduler** in the Windows Start menu.
2. Click **Create Basic Task** in the right-hand panel.
3. Set Name to `Instagram Reels Pipeline` and Trigger to **Daily** or **Hourly**.
4. In **Action**, select **Start a program**.
5. Set:
   - **Program/script**: `cmd.exe`
   - **Add arguments**: `/c "D:\Instagram\reels-pipeline\.venv\Scripts\python.exe main.py"`
   - **Start in**: `D:\Instagram\reels-pipeline`
6. Click **Finish**.

#### Linux/macOS (Cron)
Open your crontab editor:
```bash
crontab -e
```
Add the following line to run the pipeline every hour on the hour:
```cron
0 * * * * cd /path/to/reels-pipeline && .venv/bin/python main.py >> logs/cron.log 2>&1
```
The scheduling constraints (`hours_between_posts: 3` in `config.yaml`) will ensure that it only publishes a Reel once every 3 hours even if the script runs hourly.
